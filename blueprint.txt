项目内容：实现一个在线评测平台

主要对象：

* content: 一段带格式的内容，包含语言 (Markdown 等)、源码 (例如 Markdown)、渲染结果 (例如 HTML) 等信息。
* program: 用户提交的（不受信任的、需要隔离的）程序，包含语言 (C++, Python 等)、源码等信息。
* aggregator: 将评测结果 (得分, 运行时间, 运行空间等) 汇总成更大的评测结果的策略 (builtin 的 min/max/sum 或用户上传的程序 [program])。
* testcase: 一组测试数据，包括输入文件、答案文件、时间/空间限制 (optional, 若没有则采用子任务的限制) 等信息。
* subtask: 一个子任务，包括若干个测试点 [testcase]、时间/空间限制 (optional, 若没有则采用题目的限制)、信息加和方式 [aggregator]。
* problem：题目。
  * 与评测无关的有: 编号、名称、题面 [content]、可见用户组 ID、管理用户组 ID 等信息。
  * 与评测相关的有: 题目类型 (传统、提交答案、交互、自定义 (由用户上传的 [program] 定义评测逻辑))、
    包含的子任务 [subtask]、标准程序 [program]、数据校验器 [program]、答案检验器 [program]、时间/空间限制、信息加和方式 [aggregator]。
* user: 网站的用户，包括用户名、权限、等级分、贡献分等信息。
* group: 用户组，由包括组名、其包含用户 ID 等信息。
* submission: 一份提交，包括代码，所属用户 ID，所属题目 ID，评测结果等信息。
* hack: 一组针对错解构造的数据，包括输入文件、针对的提交记录 ID、评测结果等信息。
* judger: 评测机，包含可用状态等信息。
* contest: 一场比赛，包括赛制 (OI, IOI, ACM, 自定义)、题目 ID、参加用户 ID、排行榜、等级分变化系数、等级分变化等信息。
* comment：比赛或题目下的评论，包括内容 [content]，所属用户 ID，所属比赛/题目 ID，评分等信息。

对象之间的关系：

* user 与他们提交的 submission 关联。
* user 与他们参加的 contest 互相关联。
* user 与他们发布的 comment 关联。
* user 与他们提交的 hack 关联。
* problem 与它包含的 submission 关联。
* contest 与它包含的 submission 关联。
* problem/contest 与 comment 关联。

实现功能：

这个网站主要向用户提供了创建题目/比赛，并提交代码进行评测的功能。拥有权限的用户可以创建新的题目，并配置数据与时间空间限制。任何用户均可以在题目中提交自己的代码进行自动评测并返回相应的得分信息与评测细节。用户还可以对于一道题目，查看其所有提交记录，并按照一定方式筛选或排序。

用户还可以选择题库中的题目创建新的比赛。报名比赛的用户可以在规定时间内在比赛的题目中提交代码并根据不同赛制的得分规则进行排名。比赛结束后用户的等级分会依据排名进行变化。

对于题目和比赛，用户可以添加 comment 并对其它 comment 进行 upvote 或 downvote。某个用户发布的 comments 所获得的 vote 会贡献到他的贡献分上。
